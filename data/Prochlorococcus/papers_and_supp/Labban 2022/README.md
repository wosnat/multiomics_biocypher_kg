## Notes

- Gene ID format in `supp table s1.csv`: `BSR22_XXXX` (custom locus tag prefix assigned by the author pipeline). These are **not** the NCBI locus tags for RSP50 (check `cache/data/Prochlorococcus/genomes/RSP50/gene_mapping.csv` for the real NCBI prefix). An author-supplied GFF that maps `BSR22_*` to protein_id/coordinates is the only clean path to resolution.
- Strain RSP50 is already deployed in the KG (see `cyanobacteria_genomes.csv`). Only the ID-mapping layer needs to be extended.
- `supp table s1_resolved.csv` and `supp table s1_resolved_report.txt` are auto-generated artifacts — do not hand-edit.
- Two paperconfigs exist (`paperconfig.yaml` and `paperconfig_orig.yaml`) — only `paperconfig.yaml` is authoritative. The `_orig` file used `locus_tag_ncbi` as id_type; the current file uses `locus_tag`. The author-GFF bridge will bind `BSR22_*` to locus_tag.
- See `docs/methods_gene_id_mapping.md` for tier-1 ID conventions; `BSR22_*` should register as a Tier-1 gene-unique ID once the GFF is processed.
