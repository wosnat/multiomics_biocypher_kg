# Bagby and Chisholm 2015

**Citation:** Bagby SC, Chisholm SW. "Response of *Prochlorococcus* to varying CO2:O2 ratios." *The ISME Journal* 9, 2232–2245 (2015).
**DOI:** 10.1038/ismej.2015.36
**Organism(s):** *Prochlorococcus* MED4 (axenic, mid-exponential)
**Topic:** Microarray time-course (3, 6, 9, 12, 18, 24 h) of MED4 shocked with four gas mixtures (air control, −O2, −CO2, −CO2/−O2) vs pre-shock. Characterises transcriptional response to acute CO2 and O2 limitation and recovery, comparing to published high-light shock response.

## Classification

**Bucket A - fully integrated**

Four pairwise gas-vs-air DE contrasts are integrated. Table S2 cross-shock concordance (three categorical metrics, two boolean "rapid recovery" flags) was added on 2026-04-28 as two `derived_metrics_table` entries bound to the low_co2 and low_co2_no_o2 carbon experiments respectively. Source CSV is normalized via `scripts/build_modified_csv/build_bagby2015_modified_csv.py` (preamble dropped, multi-line headers collapsed, BOM stripped, en-dashes replaced). MED4 is deployed; no further work expected.

## Notes

- Gene ID format: PMM#### (old_locus_tag) + PMED4_##### (alternative_locus_tag) + probeset. `*` suffix on some locus tags marks manual CyanoBase re-curation (see moesm34 header row 4).
- Strains in/out of KG: MED4 deployed.
- `de_genes_gas_shock_vs_air.csv` is prefiltered (only significant-in-any-contrast genes) and lacks adjusted p-values; each experiment's logFC column reports the max-magnitude FC across the 3-24 h time course, so `timepoint: "max across 3-24h"` is correct and `timepoint_hours: null`.
- moesm34 contains Cyanobase functional categories (A-Z, with `[original]` re-curation annotations) that we did not promote to a derived metric (these are static ontology-style annotations, not per-condition signal). If a canonical Cyanobase ontology is added to the KG later, these labels can be re-harvested from the modified CSV's `cyanobase_category` column.
- Adapter fix shipped alongside this integration: `ObservationsAdapter.get_edges` now honors `sep` and `skip_rows` on the fallback (non-resolved) read path, mirroring the omics adapter and the validator. Bagby 2015 itself does not need this (it uses the clean `_modified.csv`), but the fix removes a class of "preamble in DM source CSV" failures for future papers.
