### S5 duplication caveat (accepted)

S5 is the *enrichment-set background population* used for the paper's KEGG/topic enrichment test on the EZ55 coculture comparison. It re-reports the same edgeR run as S3 but for a different gene slice (the 452 genes belonging to one of 8 KEGG/topic categories) and is **not** filtered to significant rows.

| Slice | S3 | S5 | Overlap |
|---|---|---|---|
| Total unique EZ55 genes | 422 | 452 | 76 |
| Genes only here | 346 | 376 | — |
| Significant rows (PValue < 0.05) | 422 (all) | 128 | ~67 (logFC matches to 4 decimals) |

Wiring all of S5 produces **duplicate `Changes_expression_of` edges** for the ~67 sig genes shared with S3 (different edge `id` from the distinct analysis id, same Experiment, same Gene). Accepted for now; revisit if duplicate edges inflate `rank_by_effect` / `significant_up_count` on the EZ55 coculture experiment. The 385 non-significant rows + ~30 sig rows missing from S3 are the new DE evidence S5 contributes; the per-category Y/blank columns are the unique-to-S5 evidence wired through the DerivedMetric path.
